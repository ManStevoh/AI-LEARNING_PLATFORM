# Institution Dashboard UX Specification

## Purpose

This document defines the institution admin dashboard for school/organization oversight.

## Admin Goals

- manage teachers and learners,
- track adoption,
- monitor learning outcomes,
- control subscription and AI usage,
- export reports,
- prepare renewal decisions.

## Dashboard Sections

- institution health summary,
- active learners/teachers,
- class progress,
- competency gaps,
- AI usage and cost,
- subscription status,
- support issues,
- reports/export center.

## Key Workflows

### Setup Checklist

Items:

- institution profile,
- subscription,
- teacher invitations,
- learner import,
- class creation,
- curriculum assignment,
- pilot success metrics.

### Analytics Review

Filters:

- date range,
- campus,
- class,
- teacher,
- level/path,
- cohort.

Metrics:

- activation,
- engagement,
- lesson completion,
- project completion,
- skill progress,
- AI usage,
- sandbox usage.

### Reports

Reports:

- class progress,
- learner progress,
- competency coverage,
- AI usage,
- subscription utilization,
- pilot outcome.

Exports must be permission-checked and audited.

## Required States

- new institution,
- no data,
- trial expiring,
- payment overdue,
- report unavailable,
- privacy threshold not met.

## Acceptance Criteria

- admin can see adoption and outcomes,
- billing and academic roles are separated,
- exports are audited,
- AI usage controls are visible,
- tenant isolation is enforced.
