# AI Mentor UX Specification

## Purpose

This document defines the user experience for AI mentor interactions.

## UX Principles

- AI should be helpful, transparent, and bounded.
- AI should scaffold before solving.
- AI should show sources when using curriculum content.
- AI should explain limits clearly.
- AI should never pretend to be human.

## Entry Points

- lesson page,
- block coding workspace,
- browser IDE,
- dashboard,
- assessment review,
- teacher dashboard,
- career roadmap.

## Mentor Panel Layout

Elements:

- conversation area,
- suggested prompts,
- source/citation area,
- mode indicator,
- quota/limit notice where relevant,
- feedback rating,
- escalate/ask teacher option.

## Modes

- Teach,
- Hint,
- Debug,
- Review,
- Quiz Me,
- Career Coach,
- Teacher Assistant.

Assessment mode:

- limited help,
- no full solutions,
- explains concepts and debugging approach.

## Trust And Evidence

AI responses should show:

- sources where used,
- assumptions,
- confidence wording when appropriate,
- tool/result used where relevant.

## Error States

- AI unavailable,
- quota reached,
- safety blocked,
- no relevant source found,
- tool failed,
- teacher/institution disabled AI.

## Acceptance Criteria

- learner can tell they are using AI,
- AI mode is visible,
- sources are shown for curriculum answers,
- users can rate/report responses,
- assessment mode restrictions are clear.
