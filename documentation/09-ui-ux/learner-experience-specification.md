# Learner Experience Specification

## Purpose

This document defines the learner-facing experience for dashboards, lessons, projects, AI guidance, progress, and portfolio.

## Experience Principles

- make the next action obvious,
- show progress without overwhelming,
- explain why recommendations are made,
- keep AI visible but not intrusive,
- support low-bandwidth and mobile layouts,
- avoid shame-based progress language.

## Learner Dashboard

Primary sections:

- today's recommended goal,
- current course/path,
- active lesson/project,
- skill progress summary,
- review due,
- AI mentor entry,
- portfolio highlights,
- streak/encouragement where appropriate.

Required states:

- first-time learner,
- no course selected,
- lesson in progress,
- blocked/prerequisite missing,
- AI quota reached,
- offline/connection warning,
- completed path.

## Lesson Experience

Lesson page should include:

- objective,
- estimated time,
- content,
- examples,
- practice,
- AI mentor panel,
- related skills,
- next step.

Rules:

- content should be chunked,
- practice should appear close to explanation,
- AI hints should use current lesson context,
- learner should always know how to continue.

## Progress View

Show:

- completed lessons,
- skills in progress,
- strengths,
- review areas,
- projects proving skills.

Avoid:

- showing hundreds of micro-competencies by default,
- using discouraging labels.

## Portfolio View

Show:

- projects,
- skills demonstrated,
- teacher approvals,
- README/demos later,
- certificate readiness.

## Acceptance Criteria

- learner can identify next action within seconds,
- learner can resume current lesson/project,
- AI recommendations include rationale,
- mobile layout remains usable,
- empty and error states are defined.
