# Level 2 To Level 5 Curriculum Roadmap

## Purpose

This document outlines the progression after Level 1 Block Coding.

The goal is to move learners from visual programming to professional software engineering through a gradual, evidence-based pathway.

## Level 2: Blocks To Code

### Goal

Learners understand that visual blocks represent real programming concepts in JavaScript and Python.

### Core Concepts

- generated code,
- sequence,
- events,
- loops,
- conditions,
- variables,
- functions,
- simple debugging,
- reading code.

### Units

1. Blocks And JavaScript.
2. Blocks And Python.
3. Events Become Functions.
4. Loops In Blocks And Code.
5. Variables And Score.
6. Conditions And Decisions.
7. Debugging Generated Code.
8. Final Blocks-To-Code Project.

### Projects

- animated story with generated code,
- score game with variables,
- simple quiz game,
- maze game with code explanation.

### Assessment

- explain generated code,
- match blocks to code,
- predict output,
- fix simple generated-code errors.

### AI Behavior

AI should:

- explain block-to-code mapping,
- avoid overwhelming syntax detail,
- compare visual and text versions,
- create extra examples.

## Level 3: Hybrid Editing

### Goal

Learners begin editing controlled parts of code while still using visual blocks.

### Core Concepts

- editable code snippets,
- syntax,
- parameters,
- functions,
- lists/arrays,
- strings,
- simple input/output,
- debugging.

### Units

1. Editing Values In Generated Code.
2. Changing Conditions.
3. Writing Simple Functions.
4. Working With Lists.
5. Debugging Syntax Errors.
6. Hybrid Game Project.
7. Hybrid Data Project.
8. Transition Reflection.

### Projects

- customizable game,
- quiz app,
- simple simulation,
- data tracker.

### Assessment

- edit code safely,
- explain syntax error,
- write small function,
- modify behavior without breaking project.

### AI Behavior

AI should:

- explain syntax in simple language,
- guide debugging,
- ask learners to predict before running,
- avoid rewriting whole project.

## Level 4: Full Coding Environment

### Goal

Learners work in a browser IDE with real files, code execution, tests, and AI mentoring.

### Core Concepts

- files and folders,
- text programming,
- terminal/output,
- tests,
- debugging,
- Git basics,
- web fundamentals,
- APIs,
- databases introduction.

### Track A: JavaScript And Web

Units:

1. JavaScript Basics.
2. HTML Structure.
3. CSS Styling.
4. DOM Interaction.
5. Forms.
6. Fetch/API Basics.
7. Local Storage.
8. Web Mini Project.

### Track B: Python Foundations

Units:

1. Python Syntax.
2. Variables And Types.
3. Conditions.
4. Loops.
5. Functions.
6. Lists And Dictionaries.
7. Files.
8. Python Mini Project.

### Projects

- portfolio page,
- calculator,
- quiz app,
- task tracker,
- weather/API app,
- Python text game.

### Assessment

- coding tasks,
- debugging challenges,
- tests,
- short explanations,
- project rubric.

### AI Behavior

AI should:

- debug with learner,
- explain test failures,
- suggest next steps,
- teach code reading,
- avoid full assignment completion.

## Level 5: Real-World Software Projects

### Goal

Learners build realistic applications using professional tools and workflows.

### Core Concepts

- Git,
- requirements,
- UI design,
- backend,
- frontend,
- database,
- authentication,
- APIs,
- testing,
- deployment,
- documentation.

### Web Engineering Track

Units:

1. Git And GitHub Basics.
2. Product Requirements.
3. Database Design.
4. Laravel Basics.
5. React Or Next.js Basics.
6. REST APIs.
7. Authentication.
8. Testing.
9. Deployment.
10. Portfolio Documentation.

Projects:

- student registration system,
- inventory system,
- school club portal,
- e-commerce mini app,
- booking system,
- learning tracker.

### Mobile Track

Units:

1. Flutter Basics.
2. Layouts And Widgets.
3. State Management.
4. Forms.
5. APIs.
6. Local Storage.
7. Authentication.
8. Mobile Capstone.

Projects:

- habit tracker,
- school notice app,
- simple marketplace,
- offline quiz app.

### AI/Data Track

Units:

1. Python For Data.
2. Data Cleaning.
3. Visualization.
4. Machine Learning Basics.
5. Prompt Engineering.
6. RAG Basics.
7. AI App Project.

Projects:

- local business data dashboard,
- AI FAQ assistant,
- simple recommendation system,
- document Q&A app.

### Cybersecurity Track

Units:

1. Web Security Basics.
2. Authentication Risks.
3. OWASP Top 10.
4. Secure Coding.
5. Basic Threat Modeling.
6. Security Testing.

Projects:

- secure login review,
- vulnerability lab,
- security checklist for a project.

### Assessment

Use:

- project rubric,
- code review,
- tests,
- deployment proof,
- README quality,
- presentation,
- interview simulation.

### AI Behavior

AI should:

- act as mentor,
- review code,
- suggest improvements,
- explain architecture,
- simulate interview,
- support portfolio polishing,
- avoid replacing learner work.

## Cross-Level Requirements

Every level must include:

- skill mapping,
- projects,
- debugging,
- retrieval practice,
- AI mentor rules,
- teacher dashboard data,
- portfolio evidence,
- accessibility considerations,
- local Kenya examples.
