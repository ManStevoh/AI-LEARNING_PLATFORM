# Level 1 Complete Content Package

## Purpose

This document defines what must exist before Level 1 Block Coding can be considered ready for pilot.

Level 1 is not only a list of topics. It must include lessons, projects, teacher guides, rubrics, AI behavior, assets, and assessment evidence.

## Target Learners

Primary:

- ages 8-12,
- complete beginners,
- school/club settings.

Secondary:

- older beginners who prefer visual programming first.

## Level 1 Outcomes

Learners should be able to:

- understand sequences,
- use events,
- use loops,
- use simple conditions,
- use variables,
- debug disconnected or misplaced blocks,
- create animations,
- create simple games,
- explain their project logic.

## Required Units

### Unit 1: Meet The Coding Studio

Content:

- interface tour,
- stage,
- sprites,
- green flag,
- first motion block.

Project:

- make a sprite move.

### Unit 2: Sequences And Stories

Content:

- order of instructions,
- animation sequence,
- speech/thought blocks.

Project:

- animated story scene.

### Unit 3: Events

Content:

- green flag,
- key press,
- sprite click.

Project:

- interactive character.

### Unit 4: Loops

Content:

- repeat,
- forever,
- wait/timing.

Project:

- dance animation or moving pattern.

### Unit 5: Conditions

Content:

- if,
- if/else,
- sensing simple state.

Project:

- choose-your-path story or simple game rule.

### Unit 6: Variables And Score

Content:

- score,
- timer,
- lives,
- changing variables.

Project:

- collect items game.

### Unit 7: Debugging

Content:

- disconnected blocks,
- wrong order,
- infinite loop,
- missing event,
- variable mistakes.

Project:

- fix broken projects.

### Unit 8: Final Project

Content:

- plan,
- build,
- test,
- explain,
- submit.

Project:

- original animation or mini-game.

## Required Assets

Each unit needs:

- lesson script,
- teacher guide,
- student instructions,
- Blockly starter project,
- sample solution,
- rubric,
- quiz/checkpoint,
- AI mentor prompt behavior,
- vocabulary list,
- offline worksheet option,
- accessibility notes.

## Teacher Guide Requirements

Each teacher guide should include:

- lesson goal,
- materials,
- timing,
- setup steps,
- teaching notes,
- common misconceptions,
- intervention tips,
- extension activity,
- assessment checklist.

## AI Mentor Behavior

AI should:

- explain in age-appropriate language,
- inspect Blockly JSON,
- give hints before answers,
- point out disconnected blocks,
- ask learner to predict behavior,
- encourage testing.

AI should not:

- complete entire final project,
- shame learners,
- overcomplicate explanations,
- expose hidden assessment answers.

## Assessment Evidence

Evidence:

- project runs,
- target blocks used,
- learner explanation,
- debugging activity,
- quiz results,
- teacher observation,
- final project rubric.

## Pilot Readiness Checklist

Level 1 is pilot-ready when:

- all eight units have lesson scripts,
- all units have Blockly starter projects,
- teacher guides exist,
- rubrics exist,
- AI prompts are reviewed,
- assets are licensed/owned,
- accessibility review is complete,
- project submission works,
- teacher dashboard can show progress.

## Acceptance Criteria

- learner can complete first project in one session,
- teacher can run lesson without platform team support,
- every project maps to competencies,
- AI hints align with lesson objectives,
- final project has clear rubric,
- pilot analytics can measure completion and struggle points.
