# Phase 1 Sprint Plan

## Purpose

This document converts Phase 1 foundation work into an initial sprint plan.

## Sprint 0: Project Setup

Goals:

- repository setup,
- Laravel scaffold,
- React/Inertia,
- PostgreSQL/Redis,
- CI,
- coding standards.

Deliverables:

- app boots locally,
- tests run,
- auth package selected,
- module structure started.

## Sprint 1: Identity And Tenancy

Goals:

- users,
- roles,
- tenant context,
- institution model,
- auth flows.

Deliverables:

- login/register,
- institution creation,
- role/permission foundation,
- tenant isolation tests.

## Sprint 2: Institution And Classes

Goals:

- class/cohort management,
- teacher invitations,
- learner enrollment.

Deliverables:

- institution admin can create class,
- teacher can view assigned class,
- learners can be enrolled.

## Sprint 3: Curriculum Foundation

Goals:

- skills,
- courses,
- lessons,
- Level 1 structure.

Deliverables:

- first lessons seeded,
- skill mapping,
- learner progress events.

## Sprint 4: Block Coding Prototype

Goals:

- Blockly workspace,
- stage runtime prototype,
- project save/load.

Deliverables:

- learner can run first project,
- project autosaves,
- generated code view prototype.

## Sprint 5: AI Mentor Foundation

Goals:

- AI Gateway,
- prompt registry,
- Teacher Agent,
- RAG over sample lessons.

Deliverables:

- AI answers lesson questions,
- usage logged,
- safety controls basic.

## Sprint 6: Teacher Dashboard And Pilot Prep

Goals:

- teacher dashboard,
- assignment flow,
- progress summary,
- pilot checklist.

Deliverables:

- teacher can assign lesson,
- teacher sees progress,
- pilot readiness review.

## Acceptance Criteria

- each sprint has demo,
- tests updated,
- docs updated,
- risks reviewed,
- pilot scope remains controlled.
