# Release Gate Checklists

## Purpose

This document defines release gates for moving from documentation to prototype, pilot, and production.

## Gate 0: Documentation Ready

Required:

- product requirements,
- architecture specs,
- AI safety specs,
- security baseline,
- roadmap,
- pending work register.

## Gate 1: Prototype Ready

Required:

- Figma prototype,
- clickable learner flow,
- block workspace prototype,
- teacher dashboard prototype,
- technical spike results.

## Gate 2: Engineering Foundation Ready

Required:

- Laravel scaffold,
- React/Inertia setup,
- PostgreSQL/Redis,
- auth,
- tenant context,
- CI tests,
- coding standards.

## Gate 3: Pilot Ready

Required:

- learner onboarding,
- class management,
- Level 1 content,
- AI mentor,
- progress tracking,
- support workflow,
- legal pilot docs,
- monitoring/backups.

## Gate 4: Paid Launch Ready

Required:

- billing/subscriptions,
- payment reconciliation,
- privacy/legal review,
- production monitoring,
- incident response,
- customer success process.

## Acceptance Criteria

- no gate is skipped,
- risks are documented,
- owners approve readiness,
- failures create action items.
