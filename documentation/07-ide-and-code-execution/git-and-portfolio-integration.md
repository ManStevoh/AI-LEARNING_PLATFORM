# Git And Portfolio Integration

## Purpose

This document defines how learner projects evolve into professional portfolio artifacts.

Git integration is important for Level 4 and Level 5 learners, but it should be introduced gradually.

## Goals

- teach version control habits,
- export projects,
- prepare GitHub-ready repositories,
- create portfolio evidence,
- map projects to competencies and careers.

## Phase 1: Internal Portfolio

Features:

- project gallery,
- project summary,
- skills demonstrated,
- teacher approval,
- AI README draft,
- screenshots/demo links later.

## Phase 2: Git Export

Features:

- download ZIP,
- export file tree,
- generate README,
- generate `.gitignore`,
- include license guidance,
- include setup instructions.

## Phase 3: GitHub Integration

Features:

- connect GitHub account,
- create repository,
- push project,
- update README,
- link repository to portfolio.

Rules:

- minors require policy review for public publishing,
- user controls external sharing,
- secrets must be scanned/blocked,
- AI-generated docs must not exaggerate learner work.

## Portfolio Evidence

Each portfolio item should include:

- project title,
- description,
- learner role,
- technologies,
- competencies demonstrated,
- screenshots,
- repository/deployment link,
- teacher/AI review status,
- reflection.

## AI Assistance

AI can help:

- write README draft,
- summarize project,
- suggest improvements,
- map skills,
- prepare demo script,
- improve documentation.

AI should not:

- claim learner built features they did not build,
- publish without user approval,
- expose private school/learner data.

## Acceptance Criteria

- learner can save project to portfolio,
- portfolio item maps to skills,
- teacher can approve where required,
- README generation is draft-only,
- external publishing is explicit,
- secrets are not exported.
