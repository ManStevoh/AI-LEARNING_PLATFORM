# Feature Catalogue

## Purpose

This catalogue lists the major product capabilities required for the AI Learning Platform.

Each feature should later receive detailed requirements, API contracts, UI specs, tests, analytics events, and ownership.

## Feature Status Labels

- Foundation: required for first serious build.
- Pilot: required for school pilot.
- Growth: required after validation.
- Future: strategic long-term capability.

## Feature Acceptance Rule

No feature should be considered ready until it has:

- product requirement,
- permission/entitlement rule,
- UI flow,
- API or backend contract,
- analytics events,
- tests,
- security/privacy review where needed.
