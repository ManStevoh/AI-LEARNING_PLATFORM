# Institution Platform

## Goal

Institutions must be first-class customers from the beginning. The product should support schools, coding academies, universities, TVET institutions, NGOs, government programs, and corporate training organizations.

## Institution Types

Supported institution categories:

- primary schools,
- junior schools,
- secondary schools,
- TVET institutions,
- universities,
- coding bootcamps,
- NGOs,
- government programs,
- corporate training departments,
- community learning centers.

## Core Roles

### Institution Owner

Can manage the institution account, billing, subscription, administrators, policies, data exports, and high-level reporting.

### Institution Administrator

Can manage campuses, teachers, learners, classes, cohorts, imports, exports, reports, and institution settings.

### Teacher

Can create classes, invite learners, assign lessons, create assessments, review submissions, view analytics, and use AI teaching tools.

### Learner

Can take lessons, build projects, ask AI questions, submit assignments, view feedback, earn badges, and track progress.

### Parent Or Guardian

Can view child progress, reports, subscriptions where applicable, and teacher messages where institution policy allows.

### Platform Super Admin

Can manage global settings, support, billing issues, content moderation, AI policies, platform analytics, and compliance operations.

## Teacher Account Creation

Teachers should be able to create student accounts through:

- manual entry,
- CSV import,
- invitation links,
- class codes,
- admin-approved bulk import,
- future SIS/LMS integrations.

For child accounts, the platform must support consent and guardian workflows based on jurisdiction and institution policy.

## Institution Analytics

Analytics should support:

- learner risk detection,
- curriculum coverage,
- teacher workload,
- intervention recommendations,
- engagement trends,
- completion rates,
- assessment performance,
- AI usage value,
- subscription ROI reports.

## Institution Subscription Requirements

Institutions should be able to buy discounted learner seats.

Subscription variables:

- number of learners,
- number of teachers,
- AI usage quota,
- content access,
- reporting level,
- support level,
- offline access,
- custom curriculum support,
- dedicated deployment option,
- country/region.

## Country And Curriculum Adaptation

The platform should support country-specific curriculum mapping starting with Kenya.

Required capabilities:

- curriculum standards database,
- grade/level mapping,
- competency mapping,
- topic mapping,
- lesson alignment,
- assessment alignment,
- teacher reports by curriculum objective,
- country-specific content packs.

## Institutional Trust Requirements

Institutions will require:

- data privacy,
- child safety,
- uptime,
- predictable pricing,
- training,
- onboarding,
- support,
- reporting,
- auditability,
- content quality,
- teacher control over AI,
- clear terms and data processing agreements.
