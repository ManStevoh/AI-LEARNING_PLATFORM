# ADR 0009: Build An Owned Design System

## Status

Accepted.

## Context

The platform needs a premium, accessible, scalable UI across learner, teacher, institution, admin, AI, block coding, and IDE experiences.

## Decision

Build an owned design system using design tokens, Tailwind, shadcn/ui, Radix UI, and accessibility-first component governance.

## Rationale

- avoids inconsistent page-by-page design,
- supports future web/mobile/desktop expansion,
- gives the product a unique brand,
- improves implementation consistency.

## Consequences

- Figma library and coded components must stay aligned,
- accessibility must be tested,
- design tokens become a source of truth.
